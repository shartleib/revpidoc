RevPi installation

Revolution Pi is able to run a wide range of software in industrial environments, including a number of different operating systems – the core software that makes a computer run. The most popular of these, and the official operating system of the Raspberry Pi Foundation, is Raspberry Pi OS. Based on Debian Linux, we’re tailor-made it for the Revolution Pi and it comes with a range of extras pre-installed and ready to go.

If you’ve only ever used Microsoft Windows or Apple macOS, don’t worry: Raspberry Pi OS is based on the same windows, icons, menus, and pointer (WIMP) principles, and should quickly feel familiar. The following chapter will get you started and introduce you to some of the bundled software.

Connect remote to your RevPi via SSH

SSH (Secure Shell) is a protocol used for securely connecting to and executing commands on remote systems, in case your RevPi. It solves the problem of potential data interception in insecure networks by encrypting all transmitted information.

To connect to your RevPi securely you only need to know its IP none or the names SN*.local

For Windows

  1. Download and install PuTTY, a free SSH client for Windows. You can download it from this URL: https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html

  2. Once installed, open PuTTY.

  3. In the 'Host Name (or IP address)' field, enter the IP address of your RevPi.

  4. Ensure that the Port is set to 22 (this is the default port for SSH).

  5. Click 'Open'.

  6. If this is your first time connecting to the RevPi from this computer, you’ll receive a warning. Click 'Yes' to continue.

  7. You will be prompted to enter your username and password for the RevPi. The default username is pi and the password is on the right side of your device.

For Mac OS

  1. Open Terminal (You can find Terminal in Applications → Utilities, or by searching for it in Spotlight).

  2. Type the following command, replacing 'your_pi_ip' with your RevPi’s IP address: ssh pi@your_pi_ip

  3. The first time you connect, you’ll see a message saying the authenticity of the host can’t be established. Type 'yes' to continue.

  4. You’ll then be prompted to enter your RevPi’s password. As with Windows, the default username is pi and the password is on the right side of your device.

For Linux

The process for Debian Linux is very similar to the process for Mac.

  1. Open a terminal.

  2. Use the SSH command to connect to your RevPi, replacing 'your_pi_ip' with your RevPi’s IP address: ssh pi@your_pi_ip

  3. Like with the Mac, you’ll see a message about the authenticity of the host the first time you connect. Type 'yes' to continue.

  4. Enter your password when prompted. The default username is pi and the password is on the right side of your device.

Connect remote to your RevPi via VNC

VNC, or Virtual Network Computing, is a graphical desktop sharing system that uses the Remote Frame Buffer protocol to remotely control another computer. It transmits the keyboard and mouse events from one computer to another, relaying the graphical screen updates back in the other direction, over a network.

The problem VNC solves is the need for remote access to a computer system’s graphical desktop environment. This can be useful in various scenarios such as technical support, remote work, or managing servers that are physically located in different places. It allows the user to interact with the remote machine as if they were sitting directly in front of it, even if the two machines are geographically far apart.

To connect to your RevPi using RealVNC, you must have the VNC Server running on your RevPi. You can enable this by going to the RevPi configuration settings (sudo raspi-config) > Interface Options > VNC > Yes.

For Windows:

  1. Download and install the VNC Viewer from the RealVNC website: https://www.realvnc.com/en/connect/download/combined/

  2. Open VNC Viewer and click on the File menu and then New Connection.

  3. In the VNC Server field, enter the IP address of your RevPi.

  4. Click OK.

  5. Now, in the main viewer window, double-click on the new connection you just created.

  6. You’ll be asked for a username and password. If you haven’t changed these on your RevPi, they’ll be "pi" and "raspberry", respectively.

  7. You’re now connected to your RevPi via RealVNC from your Windows PC.

For Mac:

  1. Download and install the VNC Viewer from the RealVNC website: https://www.realvnc.com/en/connect/download/combined/

  2. Open VNC Viewer from your Applications folder.

  3. Click on File > New Connection.

  4. In the VNC Server field, enter the IP address of your RevPi.

  5. Click OK.

  6. In the main viewer window, double-click on the new connection.

  7. You’ll be asked for a username and password. If you haven’t changed these on your RevPi, they’ll be "pi" and "raspberry", respectively.

  8. You’re now connected to your RevPi via RealVNC from your Mac.

For Debian Linux:

  1. Open your terminal.

  2. Install VNC Viewer by typing the following command and pressing Enter: sudo apt-get install realvnc-vnc-viewer

  3. After the installation, run VNC Viewer by typing vncviewer and pressing Enter.

  4. In the VNC Server field, enter the IP address of your RevPi and press Enter.

  5. You’ll be asked for a username and password. If you haven’t changed these on your RevPi, they’ll be "pi" and "raspberry", respectively.

  6. You’re now connected to your RevPi via RealVNC from your Debian Linux machine.

Check the network connection for function

nmcli is a command-line tool for controlling NetworkManager and reporting network status. It can be utilized as a replacement for nm-applet or other graphical clients. nmcli is used to create, display, edit, delete, activate, and deactivate network connections, as well as control and display network device status.

Run below ip command:

$ nmcli connection

Set the IPv4 configuration to static

By default, the Ethernet ports of the Revolution Pi are set to DHCP and get an IP from the network.

Run the nmcli command to get the connection NAME

$ nmcli connection

The result looks like this:

NAME                        UUID                                  TYPE      DEVICE
DHCP eth0                   d1ae9f0b-c8a7-42aa-a238-8f1e9668f5ef  ethernet  eth0
DHCP eth1                   4e764ae2-0a96-11ee-b8b2-97bdf6db4268  ethernet  --
Link-local eth0 (fallback)  a9883125-de1f-4d75-a049-124ee2adcff4  ethernet  --
Link-local eth1 (fallback)  02418238-fb3a-11ed-8321-c3d5d52b169a  ethernet  --

Once we get the connection NAME 'DHCP eth0'

Run below nmcli command to assign static ipv4 address

Syntax

$ nmcli con mod ‘connection-name’ ipv4.address <IP-Address>

$ nmcli con mod 'DHCP eth0' ipv4.address 192.168.1.151/24

Set the gateway IP by running the command below

$ nmcli con mod 'DHCP eth0' ipv4.gateway 192.168.1.1

Change Configuration from DHCP to Manual

This way the IP is set static and persistent on the eth0

$ nmcli con mod 'DHCP eth0' ipv4.method manual

Set the DNS server IP

DNS server translates domain names to IP addresses for the internet. Add a DNS Server IP by running below command:

$ nmcli con mod 'DHCP eth0' ipv4.dns '8.8.8.8'

Make the IPv4 changes operative

If you execute this command and are connected to the device remotely via SSH, the connection will be terminated and can no longer be activated. You must then restart the device or use the second Ethernet interface on which DHCP is still active.

Disable and enable the connection so that above changes come into the effect.

$ nmcli connection down d1ae9f0b-c8a7-42aa-a238-8f1e9668f5ef
$ nmcli connection up d1ae9f0b-c8a7-42aa-a238-8f1e9668f5ef

Result

Now Run IP Command to check IP address,

$ ip add show eth0
